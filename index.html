Process

How I build

Start with a real problem, get a working version running, then keep testing until the weak parts are obvious.

Most projects start as a question: can a bot react fast enough, can scattered notes become searchable, can an agent recover when something breaks? I use AI tools to move from idea to prototype quickly.

After that, the work is debugging, simplifying, and deciding what should be automated next. The goal is not a polished demo; it is a system that can handle live data, changing markets, and operational failures.

01 / Active

meme-sniper

A bot that autonomously trades Solana meme coins — monitoring on-chain launches, executing buys and sells, and managing exits based on configurable rules. A web app shows live positions and lets me adjust strategy without touching the code.

We wanted to understand how Solana actually works at the transaction level — meme coin launches are a useful stress test because the window is seconds wide.
